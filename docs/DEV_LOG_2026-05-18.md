## 1. Agent mode redesign (charts only, not daily bulk quotes)

### Problem

Agent mode previously treated OpenRouter like a second market provider: daily bulk quote scrape, 24h cache, `GET /api/market/stocks?refresh=1` burning tokens on every refresh. That duplicated Live/Mock and was expensive.

### Solution

| Concern | New behavior |
|---------|----------------|
| Header **Agent** | Enables agent **scrape job panel** (30-day LLM charts) |
| Quotes & news | Served from **`quoteDataMode`** (`live` \| `mock`), persisted in settings |
| `GET /api/market/stocks` in agent mode | Delegates to Live/Mock â€” **no** `getAgentBulkCached` |
| Charts on dashboard | Agent job cache (LLM 30d) if present, else Live/Mock timeseries |
| Agent news scrape on `GET /news` | Removed â€” news follows quote source |

**API:** `MarketDataSettings` includes `quoteDataMode`. `PUT /api/market/settings` accepts `{ dataMode, quoteDataMode }`. Clicking Mock/Live while in Agent updates quote source only.

**Files:** `apps/backend/src/config/marketDataMode.ts`, `apps/backend/src/modules/market/services/marketService.ts`, `packages/shared/src/types.ts`, `MarketDataProvider.tsx`, `marketApi.ts`.

### 6.1 Ground truth capture (per experiment)

When `POST /api/agent-scrape/eval/prompt/test` runs:

1. **Fetch Yahoo** â€” `yahoo-finance2` daily bars for each symbol in the experiment window (`evalWindowDays`, default 30).
2. **Build `golden[]`** â€” for each symbol: `yahooClose` (last session), `yahooPreviousClose`.
3. **Store on record** â€” `PromptEvalExperiment.golden` + `goldenReference: 'yahoo'`.
4. **Inject into prompt** â€” `goldenHint` block in `quoteScrapeAgent` so the model sees target closes (not a hard constraint â€” measured by deviation after).

This is **live ground truth at experiment time**, not the static `golden/*.json` regression files.

### 6.2 RAG pipeline (when `ragEnabled: true`)

| Step | Code | Persisted |
|------|------|-----------|
| Index | `ragService` chunks from `mockStocks` + `mockNews` | Firestore `ragChunks` (7d TTL) |
| Retrieve | `retrieveRagForSymbols(symbols, 2)` | â€” |
| Log | `writeRagRetrievalLog` | Firestore `ragRetrievalLogs` |
| Augment | `contextBlock` appended to quote agent user prompt | â€” |
| Summarize | `experiment.rag` | `chunksRetrieved`, `chunkIds`, `snippets`, `retrievalLogId` |

**Visual proof:** `RagFlowPanel` (4 steps) + detail drawer `GET /eval/prompt/:id/rag` for full snippets.

### 6.5 Link to AI predictions (indirect)

Prompt eval does **not** auto-tune `aiService` prediction prompts. The improvement loop is:


`POST /api/ai/stocks/:symbol/prediction` still uses client-supplied `historicalData[]`; quality depends on what the UI loaded (Live, Mock, or agent cache).
