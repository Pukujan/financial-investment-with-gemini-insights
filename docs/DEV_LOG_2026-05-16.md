## 1. Modular monolith restructure


## 3. Strict live mode

### Behavior

When the header toggle (or `PUT /api/market/settings`) sets **Live**:

1. Stocks, news, and charts must come from Tiingo (or stale Tiingo cache — see below).
2. Failures return **HTTP 503** with structured `code` fields — never mock catalog prices or articles.
3. Switching Mock ↔ Live calls `setMarketDataMode()` and **clears** the in-memory market cache.

### Stale cache fallback (live only)

If Tiingo is unreachable (429, network, etc.) but a previous successful bulk fetch exists:

- Quotes may be served from memory cache up to **7 days** old (`STALE_BULK_MAX_MS`).
- Response `meta.warnings` explains that data is stale.
- This is still **real Tiingo data**, not mock.

### Error codes (market)

| Code | When |
|------|------|
| `MARKET_LIVE_UNAVAILABLE` | No token, Tiingo down, bulk fetch failed with no stale cache |
| `MARKET_NEWS_FORBIDDEN` | Tiingo News 403 (common on free EOD-only plan) |
| `MARKET_NEWS_UNAVAILABLE` | Other news failures |
| `MARKET_CHART_NOT_PRELOADED` | Chart requested but symbol not in bulk preload and `TIINGO_CHART_ON_DEMAND=false` |
| `INVALID_MARKET_MODE` | Bad `dataMode` in settings API |

## 4. Chart efficiency (Tiingo rate limits)

### Problem observed

After a bulk refresh of ~100 symbols, **each dashboard chart click** triggered another `GET /tiingo/daily/{ticker}/prices`, quickly hitting Tiingo’s hourly limit (HTTP 429).

### Solution

1. **Bulk preload** — `fetchTiingoBulk()` stores both quotes and `seriesBySymbol` in one memory cache entry (`market:stocks:bulk:{mode}`).
2. **Chart endpoint** reads preloaded series first; **no API call** on chart click by default.
3. **In-flight dedupe** — `barsInFlight` Map prevents duplicate concurrent requests for the same symbol during bulk.
4. **Tighter date range** — `startDate` / `endDate` span ~32 calendar days (~22 trading days), not full history.
5. **Slower batching** — defaults `TIINGO_BATCH_SIZE=3`, `TIINGO_BATCH_DELAY_MS=500` to stay under free-tier limits.
6. **`TIINGO_CHART_ON_DEMAND=false`** (default) — explicit opt-in for per-click API calls (debug only).

### Meta on chart responses

`getTimeSeriesMeta(symbol)` can expose `chartSource`, `syntheticChart`, `chartNote` for UI hints when data is from preload vs on-demand.

## 5. Tiingo News integration

### Backend

- `fetchTiingoMarketNews()` — `GET /tiingo/news` with `TIINGO_NEWS_LIMIT`.
- Validates response is an array; maps Tiingo items → `NewsArticle`.
- `isTiingoNewsForbidden()` detects 403 / “News API” messages.
- Cached 24h in memory (same TTL as quotes via `MARKET_CACHE_TTL_HOURS`).
- `probeTiingoProvider()` checks EOD + optional news availability for settings probe.

### Free-tier reality

Tiingo’s **EOD token often does not include News API**. Live news returns:

```json
{
  "success": false,
  "error": "Live news requires Tiingo News API (not included on free EOD plan)...",
  "code": "MARKET_NEWS_FORBIDDEN"
}
```

### Frontend behavior (live)

- `MarketDataProvider` loads stocks first; news fetch is **isolated** in try/catch.
- Stock grid still renders if news fails.
- `newsError` / `newsErrorCode` exposed separately from stock `error`.
- `NewsFeed` shows `StatusBanner` + empty-state hint when live news is empty.

## 6. AI insights — validation and strict mode

### New / updated backend files

| File | Purpose |
|------|---------|
| `insightsValidation.ts` | Validates AI JSON shape before returning to client |
| `insightsCacheService.ts` | `getAIInsightsWithMeta()`, cache bypass via `?refresh=1` |
| `aiService.ts` | `strict` option — no `generateMockInsights()` on failure in live |

### Validation rules (`validateAIInsights`)

Requires non-empty:

- `recommendations`, `trends`, `risks`
- `portfolio.diversificationAdvice`
- `stats.accuracyRate`

Each recommendation must have `symbol`, `action`, numeric `confidence`.

Failures throw `AppError` with code `AI_INVALID_RESPONSE` (502).

### Strict live flow (`getAIInsightsWithMeta`)

1. Optional Firestore cache hit (15 min TTL) unless `bypassCache`.
2. Load stocks via `marketService.getAllStocks()` — fail with `AI_INSUFFICIENT_MARKET_DATA` if empty.
3. In live mode, attempt news; **news 503** becomes a **warning**, not a hard failure (insights proceed with stocks only).
4. Call `generateAIInsights(..., { strict: true })` in live mode.
5. Return `{ insights, meta }` with `stocksAnalyzed`, `newsArticlesUsed`, `warnings`.

### AI error codes

| Code | When |
|------|------|
| `AI_NOT_CONFIGURED` | No `OPENROUTER_API_KEY` in live mode |
| `AI_INSUFFICIENT_MARKET_DATA` | Zero stocks loaded |
| `AI_INVALID_RESPONSE` | Model returned incomplete/invalid JSON |
| `AI_GENERATION_FAILED` | OpenRouter/network/parse failure |
| `MARKET_DATA_UNAVAILABLE` | Could not load stocks for insights |

### API

- `GET /api/ai/insights` — body in `data`, metadata in `meta`
- `GET /api/ai/insights?refresh=1` — bypass Firestore cache

### Tests

- `insightsValidation.test.ts` — unit tests for validator
- QA test updated to mock `getAIInsightsWithMeta` (not deprecated `getAIInsights`)
- QA test for news 503 / `MARKET_NEWS_FORBIDDEN`

## 10. Known limitations and follow-ups

1. **Tiingo free tier** — ~1000 requests/day, hourly caps; bulk + chart discipline required.
2. **Tiingo News** — typically requires paid News API; live news will 403 on EOD-only plans.
3. **Outdated docs** — `docs/MARKET_DATA.md`, `docs/CACHE.md`, `docs/ARCHITECTURE.md`, `AGENTS.md`, and `README.md` still mention Yahoo in places. **`docs/HOW_IT_WORKS_NOW.md`** is the up-to-date operational reference.
4. **Firestore permissions** — portfolio/AI cache writes may log `PERMISSION_DENIED` if Firebase rules are not configured; app degrades to in-memory behavior.
5. **Optional future work** — `newsValidation.ts` for per-article field checks; TanStack Query on frontend; split stale news cache from quotes.
