## 3. Strict live mode

### Behavior

When the header toggle (or `PUT /api/market/settings`) sets **Live**:

1. Stocks, news, and charts must come from Tiingo (or stale Tiingo cache â€” see below).
2. Failures return **HTTP 503** with structured `code` fields â€” never mock catalog prices or articles.
3. Switching Mock â†” Live calls `setMarketDataMode()` and **clears** the in-memory market cache.

### Stale cache fallback (live only)

If Tiingo is unreachable (429, network, etc.) but a previous successful bulk fetch exists:

- Quotes may be served from memory cache up to **7 days** old (`STALE_BULK_MAX_MS`).
- Response `meta.warnings` explains that data is stale.
- This is still **real Tiingo data**, not mock.

### Error codes (market)

| Code | When |
|------|------|
| `MARKET_LIVE_UNAVAILABLE` | No token, Tiingo down, bulk fetch failed with no stale cache |
| `MARKET_NEWS_FORBIDDEN` | Tiingo News 403 (common on free EOD-only plan) |
| `MARKET_NEWS_UNAVAILABLE` | Other news failures |
| `MARKET_CHART_NOT_PRELOADED` | Chart requested but symbol not in bulk preload and `TIINGO_CHART_ON_DEMAND=false` |
| `INVALID_MARKET_MODE` | Bad `dataMode` in settings API |
| `AGENT_NOT_CONFIGURED` | Agent mode without `OPENROUTER_API_KEY` |
| `AGENT_SCRAPE_FAILED` | Agent scrape returned no quotes or job failed |
| `AGENT_SCRAPE_TIMEOUT` | Job exceeded wall-clock limit |

## 4. Chart efficiency (Tiingo rate limits)

### Problem observed

After a bulk refresh of ~100 symbols, **each dashboard chart click** triggered another `GET /tiingo/daily/{ticker}/prices`, quickly hitting Tiingoâ€™s hourly limit (HTTP 429).

### Solution

1. **Bulk preload** â€” `fetchTiingoBulk()` stores both quotes and `seriesBySymbol` in one memory cache entry (`market:stocks:bulk:{mode}`).
2. **Chart endpoint** reads preloaded series first; **no API call** on chart click by default.
3. **In-flight dedupe** â€” `barsInFlight` Map prevents duplicate concurrent requests for the same symbol during bulk.
4. **Tighter date range** â€” `startDate` / `endDate` span ~32 calendar days (~22 trading days), not full history.
5. **Slower batching** â€” defaults `TIINGO_BATCH_SIZE=3`, `TIINGO_BATCH_DELAY_MS=500` to stay under free-tier limits.
6. **`TIINGO_CHART_ON_DEMAND=false`** (default) â€” explicit opt-in for per-click API calls (debug only).

## 8. Agent scrape mode (OpenRouter)

### Purpose

Third data path when `MARKET_DATA_MODE=agent` or header toggle **Agent**. Uses LLM agents to produce structured **quotes** and **news** without consuming Tiingo quota. Prices are **model-estimated**, not exchange feeds.

### Architecture

```
Header: Mock | Live | Agent
    â†“
MarketDataProvider (agent mode)
    â†“
AgentScrapePanel â€” estimate â†’ Start | Load cached
    â†“
POST /api/agent-scrape/jobs  â†’  background job (agentScrapeJobService)
    â†“
quoteScrapeAgent (batches of 5) â†’ optional chartScrapeAgent (1 symbol/call) â†’ newsScrapeAgent
    â†“
Memory cache (batch keys + bulk key) â†’ GET /api/market/stocks serves cached agent bulk
```

### Key backend modules

| Path | Role |
|------|------|
| `modules/agent-scrape/services/agentScrapeService.ts` | Quote/news fetch, bulk cache, `timeSeriesFromQuote()` synthetic charts |
| `modules/agent-scrape/services/agentScrapeJobService.ts` | Job queue, steps, retries, timeouts, usage aggregation |
| `modules/agent-scrape/services/agents/quoteScrapeAgent.ts` | LLM â†’ `StockQuote[]` JSON |
| `modules/agent-scrape/services/agents/chartScrapeAgent.ts` | LLM â†’ 30-day OHLC per symbol |
| `modules/agent-scrape/services/agents/newsScrapeAgent.ts` | LLM â†’ `NewsArticle[]` JSON |
| `modules/agent-scrape/services/agentScrapeCache.ts` | Per-batch + bulk + news cache keys |
| `modules/ai-estimate/` | Token/cost **estimates** (no scrape LLM) |

### Job steps (example: 20 symbols, charts on)

1. **Quotes** â€” 4 steps Ă— 5 symbols (`batch-0` â€¦ `batch-3`)
2. **Charts** â€” 20 steps Ă— 1 symbol (`chart-batch-0` â€¦ `chart-batch-19`) when `scrapeCharts=true`
3. **News** â€” 1 step

Progress appears in the floating **Agent queue** (bottom-right); state persists in `localStorage` (`investai-agent-queue-v1`).

### When OpenRouter is actually called

| Action | OpenRouter chat? | Notes |
|--------|------------------|-------|
| Switch to Agent / load estimate | **No** (scrape) | `GET /api/ai-estimate/agent-scrape` uses heuristics + cache inspection |
| Estimate pricing | Optional `GET /v1/models` | Catalog cached ~1 hour for USD cost |

| **Load cached** | **No** | Serves memory cache only |

## 9. Chart scrape â€” JSON failures and fixes

### Problem

With **Scrape 30-day charts** enabled, chart steps failed at ~89% job progress:

```
Expected ',' or ']' after array element in JSON at position 17482
```

Quotes batches (5 symbols, ~500 tokens each) succeeded; chart batches (5 symbols Ă— 30 OHLC points) produced **oversized JSON** that was often **truncated** mid-array.

### Root causes

1. Five symbols per chart LLM response â†’ ~15k+ character JSON, often cut off at `max_tokens: 8192`.
2. `parseJsonFromText` only stripped markdown fences; no repair for truncated arrays.
3. Eval records were built **before** `job.completedAt` was set â†’ server-side `buildEstimateEval` returned `null`.

### Fixes shipped (`ee7603f`)

| Change | Detail |
|--------|--------|
| `AGENT_SCRAPE_CHART_BATCH_SIZE=1` | One symbol per chart step and per LLM call |
| Compact chart JSON | `{"symbol":"AAPL","bars":[["YYYY-MM-DD",o,h,l,c],...]}` |
| `max_tokens: 2048` per chart symbol | Down from 8192 shared across 5 symbols |
| `salvageTruncatedJson()` in `aiClient.ts` | Close truncated arrays/objects when possible |
| Chart step fallback | On failure, `timeSeriesFromQuote()` for that symbol; job can still complete |
| Eval recording order | Set `completedAt` **before** `buildEstimateEval` / `buildChartEval` |

## 10. Estimate eval dashboard

### Purpose

Compare **pre-run token/cost estimate** (snapshot at job start) vs **actual OpenRouter usage** after job completes â€” â€śgolden dataâ€ť for calibrating heuristics in `agentScrapeTokens.ts`.

### Data model (`packages/shared/src/estimateEval.ts`)

- `AgentEstimateSnapshot` â€” stored on job as `estimateSnapshot`
- `AgentEstimateEvalRecord` â€” per completed job: deltas, accuracy rating (`excellent` â€¦ `cached`)
- `AgentEstimateEvalHistory` â€” `records[]` + `summary` (avg deltas, accuracy mix)

### APIs

| Endpoint | Purpose |
|----------|---------|
| `GET /api/ai-estimate/agent-scrape?scrapeCharts=1` | Pre-scrape cost estimate (frontend) |
| `GET /api/agent-scrape/eval/estimates` | Eval history (merged sources) |

### Persistence (fix for â€śonly 1 row in historyâ€ť)

| Store | Behavior |
|-------|----------|
| **Server memory** | `recordEstimateEval()` array (lost on restart without disk) |
| **`apps/backend/.data/estimate-eval-history.json`** | Written on each record; loaded at startup (gitignored) |
| **Recent jobs** | API merges disk history + eval rebuilt from last 50 jobs in memory |
| **Browser `investai-estimate-eval-v1`** | Up to 50 records in `localStorage` |
| **Queue prefs `completedEvals`** | Appended on each completed job in `persistAgentJob` |

### Frontend

- Nav: **Estimate eval** â†’ `EstimateEvalDashboard.tsx`
- Merges API + local via `collectAllLocalEstimateEvals()`
- **Latest scrape** comparison card + full **History** table

### Accuracy ratings

| Rating | Rule |
|--------|------|
| `cached` | `tokensUsed === 0` |
| `excellent` | â‰¤10% token delta |

## 16. Known limitations

1. **Tiingo free tier** â€” hourly/daily caps; live mode needs discipline on bulk + charts.
2. **Tiingo News** â€” often 403 on EOD-only plans.
3. **Agent data** â€” LLM-estimated; not suitable for trading decisions.
4. **Job / eval server state** â€” in-memory jobs pruned to 10; eval history survives restart via `.data/` file.
5. **Browser eval history** â€” per-browser `localStorage`; not synced across devices.
6. **Firestore** â€” portfolio/AI cache may log `PERMISSION_DENIED` without rules; degrades gracefully.
